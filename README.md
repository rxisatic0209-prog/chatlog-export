# chatlog-export

基于 [sjzar/chatlog](https://github.com/sjzar/chatlog) 做的导出增强版，重点补了下面这些场景：

- 终端 UI 里直接搜索并选择群聊导出
- 导出支持日期范围筛选
- 导出结果默认落到桌面，方便普通用户查找
- 提供 `chatbrowser.html`，方便直接浏览导出的 JSON
- 保留原项目的 HTTP / MCP 能力，方便接到 agent 或桌面 AI 工具

## 目前这版的重点改动

- 导出聊天对象时只列群聊，不再混入私聊联系人
- “全部导出”现在是“全部群聊记录”
- “导出我的发言”也只针对群聊
- 导出选项新增：
  - `开始日期`
  - `结束日期`
- TUI 导出默认保存到桌面：
  - `~/Desktop/export_json`
  - `~/Desktop/export_csv`

## 快速使用

### 1. 运行程序

如果你已经有编译好的二进制，直接运行即可。

如果你要自己编译：

```bash
go build -o chatlog_mac .
./chatlog_mac
```

### 2. 获取密钥并解密

主菜单里的推荐顺序：

1. `获取数据密钥`
2. `解密数据`
3. `导出聊天记录`

如果自动获取密钥失败，也可以走：

1. `设置`
2. `设置数据密钥`
3. 手动填入 key
4. 再执行 `解密数据`

### 3. 导出

当前导出流程支持：

- 选择单个群聊
- 导出全部群聊
- 导出自己在群聊中的发言
- 选择导出格式：`json` / `csv`
- 选择是否导出图片
- 按日期范围筛选

日期格式使用：

```text
YYYY-MM-DD
```

例如：

```text
开始日期：2026-03-01
结束日期：2026-03-09
```

留空表示不限。

## 导出文件保存位置

TUI 导出默认保存到桌面：

- JSON: `~/Desktop/export_json`
- CSV: `~/Desktop/export_csv`

如果勾选导出图片，图片会放在对应导出文件同目录下的同名文件夹中。

## JSON 浏览器

仓库自带：

- [export_json/chatbrowser.html](./export_json/chatbrowser.html)

用途：

- 直接打开导出的 JSON
- 按聊天气泡形式浏览消息
- 支持基础搜索
- 支持图片目录选择

### 当前浏览器支持情况

支持较好的类型：

- 文本
- 图片
- 基础 emoji 文本
- 表情消息的部分占位展示
- 分享 / 引用消息的简化展示

当前仍然是“简化展示”的类型：

- 小程序卡片
- 自定义表情包 / GIF 表情
- 语音 / 视频
- 其他微信特殊消息

这类消息通常会退化为：

- 类型占位文本
- 原始文本
- 简化卡片

## MCP / Agent 集成

这个项目已经带 MCP 能力，不需要额外改造成 MCP 工具。

启动 HTTP 服务后，可通过：

- `http://127.0.0.1:5030/sse`

接入支持 MCP SSE 的客户端。

详细说明见：

- [docs/mcp.md](./docs/mcp.md)

注意：

- 默认只监听本机 `127.0.0.1`
- 当前版本没有 token 鉴权
- 如果要长期给 agent 用，建议后续补鉴权

## 常见问题

### 1. 获取数据密钥失败：`no valid key found`

这通常不是导出逻辑的问题，而是当前机器没能自动提取到微信 key。

常见原因：

- 当前机器环境和开发机不同
- 微信版本虽然相同，但内存布局不同
- macOS 上自动取 key 本来就比较脆弱

可选处理方式：

- 先在用户自己的机器上尝试自动获取
- 自动失败时，手动填 key
- 或者只分享已经解密后的数据 / 已导出的 JSON

### 2. 为什么导出文件不在程序目录？

这版 TUI 默认导出到桌面，不再跟随启动目录。

### 3. 为什么导出列表只显示群聊？

这是当前版本的设计目标：导出流程只服务群聊导出场景。
